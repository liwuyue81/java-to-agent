⏺ logging.basicConfig() 是什么

  配置日志的全局设置，类比 Java 里的 logback.xml 或 log4j.properties。

  ---
  逐参数解释

  level=logging.INFO

  设置日志级别，低于这个级别的日志不输出：

  DEBUG < INFO < WARNING < ERROR < CRITICAL

  level=INFO → DEBUG 不输出，INFO 及以上都输出

  类比 logback.xml：
  <root level="INFO">

  ---
  format="%(asctime)s [%(levelname)s] %(name)s - %(message)s"

  定义每条日志的输出格式，每个占位符对应一个字段：

  ┌───────────────┬─────────────┬─────────────────────────┐
  │    占位符     │    含义     │        输出示例         │
  ├───────────────┼─────────────┼─────────────────────────┤
  │ %(asctime)s   │ 时间戳      │ 2024-01-01 10:00:00,123 │
  ├───────────────┼─────────────┼─────────────────────────┤
  │ %(levelname)s │ 日志级别    │ INFO / ERROR            │
  ├───────────────┼─────────────┼─────────────────────────┤
  │ %(name)s      │ logger 名称 │ alert.monitor           │
  ├───────────────┼─────────────┼─────────────────────────┤
  │ %(message)s   │ 日志内容    │ 读取到5条新日志         │
  └───────────────┴─────────────┴─────────────────────────┘

  实际输出效果：
  2024-01-01 10:00:00,123 [INFO] alert.monitor - 读取到5条新日志
  2024-01-01 10:00:01,456 [ERROR] rag.log_indexer - 索引失败

  类比 logback.xml：
  <pattern>%d{yyyy-MM-dd HH:mm:ss} [%level] %logger - %msg%n</pattern>

  ---
  和 getLogger 的关系

  # main_rag.py 里
  logging.basicConfig(...)        # 全局配置，整个程序生效一次

  # 各模块里
  logger = logging.getLogger(__name__)   # 每个模块拿自己的 logger
  logger.info("读取到5条新日志")          # 输出时套用 basicConfig 的格式

  basicConfig 是装修房子，getLogger 是每个房间的开关，格式统一由 basicConfig 控制。

  ---
  model: str 是什么

  这是 Pydantic 的字段定义语法：

  # Pydantic 字段定义
  model: str                        # 必填字段，类型是 str
  temperature: float = 0            # 可选字段，默认值 0
  reasoning: bool | str | None = None  # 可选字段，默认值 None

  类比 Java + Lombok：
  @Data
  public class ChatOllama {
      private String model;               // 必填
      private float temperature = 0;      // 有默认值
      private Object reasoning = null;    // 有默认值
  }

  Pydantic 会自动读这些字段，生成构造方法，所以你不会看到显式的 __init__。

  ---
  bool | str | None 是什么

  Python 3.10+ 的类型联合写法，表示这个字段可以是多种类型：

  bool | str | None
  # 等价于旧写法：Optional[Union[bool, str]]

  类比 Java：
  // Java 没有直接等价，通常用 Object 或泛型处理
  Object reasoning = null;

  None 是 Python 的 null。

  ---

  model: str                    # 没有默认值 → 必填
  temperature: float = 0        # 有默认值   → 可选
  reasoning: bool | str | None = None  # 有默认值 → 可选

  规则：有 = 的是可选，没有 = 的是必填。

❯ 关键字传参和位置传参是什么                                                                                                                                                         

⏺ 位置传参

  按顺序匹配参数，不写参数名：

  def add(a, b, c):
      return a + b + c

  add(1, 2, 3)
  # a=1, b=2, c=3  按位置顺序匹配

  类比 Java，Java 只有位置传参：
  int add(int a, int b, int c) { ... }
  add(1, 2, 3);  // Java 只能这样

  ---
  关键字传参

  写参数名，顺序无所谓：

  def add(a, b, c):
      return a + b + c

  add(c=3, a=1, b=2)   # 顺序打乱也没问题
  # a=1, b=2, c=3

  Java 没有这个语法。

  ---
  两种混用

  add(1, b=2, c=3)   # 第一个位置传，后面关键字传
  # 规则：位置传参必须在关键字传参前面
  add(a=1, 2, 3)     # 错误！关键字传参后面不能有位置传参

  ---
  * 强制关键字传参

  def pull(owner_repo_commit, *, api_key=None):
      pass

  pull("hwchase17/react-chat", api_key="xxx")   # 正确
  pull("hwchase17/react-chat", "xxx")           # 错误，* 后面必须关键字传参

  ---
  为什么要有关键字传参

  参数多的时候，关键字传参可读性更好：

  # 位置传参，看不出每个值是什么意思
  ChatOllama("qwen2.5:7b", 0, 60, True, False)

  # 关键字传参，一眼看懂
  ChatOllama(model="qwen2.5:7b", temperature=0, timeout=60)

❯ python定义的方法参数，不需要全部传是吗，这个java不一样                                                                                                                             

⏺ 对，这是 Python 和 Java 的一个重要区别。

  ---
  Python：有默认值的参数可以不传

  def greet(name: str, prefix: str = "Hello", punct: str = "!"):
      return f"{prefix}, {name}{punct}"

  greet("Tom")                    # 只传必填，Hello, Tom!
  greet("Tom", prefix="Hi")       # 传部分，Hi, Tom!
  greet("Tom", "Hi", ".")         # 全传，Hi, Tom.

  规则：有默认值的参数可以不传，没有默认值的必须传。

  ---
  Java：必须全部传，否则用重载解决

  // Java 没有默认值，要提供多个重载
  String greet(String name) {
      return greet(name, "Hello", "!");
  }
  String greet(String name, String prefix) {
      return greet(name, prefix, "!");
  }
  String greet(String name, String prefix, String punct) {
      return prefix + ", " + name + punct;
  }

  Python 一个方法就解决了，Java 要写三个。

  ---
  对应到你项目的代码

❯ 你说的len()能传不同类型的参数，这也是python的函数特性吗                                                                                                                            

⏺ 对，这是 Python 的鸭子类型特性。

  ---
  什么是鸭子类型

  ▎ "如果它走路像鸭子，叫声像鸭子，那它就是鸭子。"

  Python 不关心变量是什么类型，只关心它有没有对应的能力：

  # len() 的逻辑：
  # 不管你传什么类型，只要你有 __len__ 方法，我就能算出长度

  len([1, 2, 3])   # list 有 __len__ → 返回 3
  len("hello")     # str  有 __len__ → 返回 5
  len({"a": 1})    # dict 有 __len__ → 返回 1
  len(123)         # int  没有 __len__ → 报错

  ---
  对比 Java 的做法

  Java 是强类型，不同类型必须调不同方法：

  list.size()      // List 专用
  str.length()     // String 专用
  map.size()       // Map 专用

  // Java 没办法写一个方法同时接受 List、String、Map

  Python 可以：
  def print_length(container):
      print(len(container))   # 管你是啥类型，有 __len__ 就行

  print_length([1, 2, 3])    # 3
  print_length("hello")      # 5
  print_length({"a": 1})     # 1

  ---
  Java 最接近的概念

  类比 Java 的接口：

  // Java 靠接口约束能力
  interface Sizeable {
      int size();
  }
  // 只有实现了 Sizeable 的类才能传进来

  // Python 靠鸭子类型约束能力
  // 只要有 __len__ 方法就行，不需要显式实现接口

  ---
  一句话总结

  Java：关心你是什么类型
  Python：只关心你有没有我需要的能力
