# Prompt 测试指南

## 核心思想

Prompt 改了就要测，测了才知道好不好。类比代码改了要跑单测。

---

## 三种测试方法

### 方法1：人工抽查（最快，适合早期）

**适用场景**：刚写完 Prompt，快速验证方向对不对。

**做法**：
1. 准备 10-20 个真实案例
2. 逐条人工判断输出是否符合预期
3. 记录失败案例，分析共同原因
4. 修改 Prompt，重新验证

**判断标准**：
```
✅ 回答准确，格式符合要求，没有幻觉
❌ 回答跑题 / 格式错误 / 编造内容 / 遗漏要求
```

---

### 方法2：准确率测试（量化，适合对比两版 Prompt）

**适用场景**：有了两版 Prompt，需要数据决定用哪个。

**做法**：

```python
# 准备测试集
test_cases = [
    {
        "input": "2024-01-01 ERROR UserService - DB connection timeout",
        "expected": "数据库连接超时"   # 标准答案
    },
    {
        "input": "2024-01-01 ERROR PaymentService - Redis unavailable",
        "expected": "Redis 不可用"
    },
    # ... 准备 50-100 条
]

# 跑测试
correct = 0
for case in test_cases:
    result = llm.invoke(prompt_template.format(log=case["input"]))
    if case["expected"] in result.content:
        correct += 1

accuracy = correct / len(test_cases)
print(f"准确率：{accuracy:.1%}")   # 例如：准确率：85.0%
```

**对比流程**：
```
Prompt A → 跑100条 → 准确率 82%
Prompt B → 跑100条 → 准确率 91%
→ 选 Prompt B
```

---

### 方法3：LLM 评分（用模型评模型，适合大批量）

**适用场景**：测试集大，人工评估成本高。

**做法**：

```python
eval_prompt = """
请评价以下回答的质量，从1-10分打分：

问题：{question}
标准答案：{ground_truth}
模型回答：{model_answer}

评分维度：
- 准确性（5分）：是否和标准答案一致
- 完整性（3分）：是否完整回答了问题
- 简洁性（2分）：是否简洁不啰嗦

只输出 JSON：
{
  "score": 总分,
  "reason": "扣分原因"
}
"""

# 批量评估
scores = []
for case in test_cases:
    score_result = eval_llm.invoke(eval_prompt.format(...))
    scores.append(json.loads(score_result.content)["score"])

avg_score = sum(scores) / len(scores)
print(f"平均得分：{avg_score:.1f} / 10")
```

**注意**：评分 LLM 本身也可能出错，需要人工抽查 10-20% 的评分结果。

## 快速判断 Prompt 好坏的检查清单

不需要每次都跑测试，日常写完 Prompt 自检：

```
□ 角色是否明确？
□ 任务是否单一？（一个 Prompt 只做一件事）
□ 约束是否具体？（"简短"改成"3句话以内"）
□ 输出格式是否固定？（给了示例或模板）
□ 边界情况是否处理？（不确定时怎么回答）
□ 是否有防幻觉约束？（"只基于提供内容"）
```

---

## 实际测试流程

```
第一步：写第一版 Prompt（凭经验，5分钟）
    ↓
第二步：人工跑 10-20 个真实案例（30分钟）
    ↓
第三步：记录失败案例，找共同原因，修改 Prompt
    ↓
第四步：准备测试集（50-100条），量化对比改前改后
    ↓
第五步：上线，持续监控业务指标
    ↓
第六步：发现问题 → 回到第三步
```

---

## 业务指标（最终标准）

Prompt 准确率高不等于业务指标好，最终看：

| 场景 | 业务指标 |
|---|---|
| 日志告警分析 | 根因定位准确率、误报率 |
| 客服问答 | 用户满意度、转人工率 |
| 代码生成 | 代码跑通率、测试通过率 |
| 知识库问答 | 答案来源准确率、幻觉率 |

---

## 常见陷阱

**陷阱1：测试集和生产数据分布不一致**
```
测试集全是简单日志 → 准确率 95%
生产遇到复杂多服务日志 → 准确率 60%
```
解决：测试集要覆盖边界情况和复杂案例。

**陷阱2：只测成功路径**
```
一定要测：
  - 日志为空时怎么回答
  - 日志格式异常时怎么回答
  - 问题超出知识范围时怎么回答
```

**陷阱3：过拟合测试集**
```
反复针对测试集调 Prompt → 测试集准确率100%
换一批新数据 → 准确率下降
```
解决：测试集和调优用的案例要分开。
